03
Key Features
🗺️
Procedural Spawner
Chests spawn dynamically each session via Physics Raycasting to ground them on uneven terrain — no two playthroughs identical.
→ Physics.Raycast · Random Positioning
🌫️
Cursed Zone System
Localized URP Volume Overrides apply a green tint + vignette within 5 m of each chest — guiding players through the environment, not UI markers.
→ URP Local Volumes · Color Grading
🎮
Raycast Interaction
Camera forward vector detects chests in range. Press [E] to collect. Detection logic is fully decoupled from player movement.
→ Unity Input System · Raycast
🧠
GameManager Singleton
Central state manager synchronises chest counter, countdown timer and win/loss conditions — one reliable source of truth.
→ Singleton Pattern · State Machine

05
Design Decisions
WHY?
Procedural vs Fixed Placement

Fixed chests are solvable after one run. Procedural generation keeps exploration meaningful every session.

WHY?
Environmental Cues vs Minimap

Minimaps reduce immersion. Green fog forces physical exploration and keeps players inside the world.

WHY?
5-Minute Timer as Core Pressure

Short sessions make every movement meaningful — explore further or rush to the nearest chest?

WHY?
Singleton GameManager

Centralised state eliminates sync bugs between UI, spawner and timer — critical for reliable win/loss validation.
